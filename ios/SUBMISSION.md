# TestFlight & App Store submission — the walkthrough (Chunk 10)

The companion to [`APP_STORE.md`](./APP_STORE.md) (which holds the
paste-ready listing material: name, subtitle, age rating, privacy
table, screenshot list, description). This file is the **how**: from
"I have a Mac and $99" to "the app is on the App Store," in order,
with the exact text to paste when Apple asks questions.

> **The two things that can actually block you:**
> 1. **The privacy policy URL.** App Store submission requires one and
>    the site doesn't have a standalone `/privacy` page yet (it's a
>    footer modal). Internal TestFlight does NOT need it — you can do
>    steps 1–5 today and fix this before step 7.
> 2. **Stripe is LIVE.** The app talks to the production backend with
>    live Stripe keys. Any checkout you "test" is a real charge — see
>    the boxed note in step 5 for the right way to test it.

---

## 1. Enroll in the Apple Developer Program ($99/year)

1. Go to [developer.apple.com/programs/enroll](https://developer.apple.com/programs/enroll)
   and sign in with the Apple ID you want the store account tied to.
2. Choose the entity type:
   - **Individual** — fastest (usually < 48 h). The App Store "seller"
     line will show **your personal legal name**, not "Lusik & Sons."
   - **Organization** — shows the business name as seller, but needs a
     legal entity (LLC) + a D-U-N-S number, and verification can take
     1–2 weeks. If Lusik & Sons is (or becomes) an LLC, this reads
     more professional on the store page.
   Either works; you can't easily switch later, so pick deliberately.
3. Pay the $99/yr fee and wait for the "Welcome to the Apple Developer
   Program" email before continuing.

## 2. Wire up signing (one time)

1. On your Mac: **Xcode → Settings → Accounts → "+" → Apple ID** and
   sign in with the enrolled account.
2. Find your **Team ID**: [developer.apple.com/account](https://developer.apple.com/account)
   → Membership details → Team ID (10 characters, like `ABCDE12345`).
3. Put it in `ios/project.yml` — uncomment the `DEVELOPMENT_TEAM` line
   and paste your ID. This matters because the Xcode project is
   regenerated by `xcodegen`: a team picked only in Xcode's UI would
   be wiped on the next regeneration; in `project.yml` it survives.
4. Regenerate + open: `cd ios && xcodegen && open LusikSons.xcodeproj`.
5. Select the LusikSons target → Signing & Capabilities → check
   **"Automatically manage signing"** shows your team with no errors.
   Xcode will create the signing certificate and register the bundle
   id (`com.lusikandsons.app`) on first use — nothing to do manually.

## 3. Create the app record in App Store Connect

1. [appstoreconnect.apple.com](https://appstoreconnect.apple.com) →
   **Apps → "+" → New App**.
2. Fill exactly:
   - Platform: **iOS**
   - Name: **Lusik & Sons** (if taken, "Lusik & Sons — Armenian Gifts")
   - Primary language: **English (U.S.)**
   - Bundle ID: **com.lusikandsons.app** (appears in the dropdown after
     step 2's first build/archive; if it's missing, register it at
     developer.apple.com/account → Identifiers → "+" → App IDs)
   - SKU: **lusiksons-ios-001**
3. That's it for now — the listing content comes in step 7.

## 4. Archive and upload a build

1. Before EVERY upload: bump `CURRENT_PROJECT_VERSION` in
   `ios/project.yml` (1 → 2 → 3 …; Apple rejects a reused build
   number), run `xcodegen`, reopen the project.
   `MARKETING_VERSION` stays `0.1.0` for TestFlight; set it to
   `1.0.0` for the build you intend to submit (matches APP_STORE.md's
   pre-flight).
2. In Xcode, set the run destination to **Any iOS Device (arm64)** —
   not a simulator, or Archive is grayed out.
3. **Product → Archive** (a few minutes). The Organizer window opens
   with the archive.
4. **Distribute App → App Store Connect → Upload**, accept the
   defaults (automatic signing, upload symbols). Xcode validates and
   uploads.
5. Wait for the "build finished processing" email (~5–30 min). The
   build then appears in App Store Connect → TestFlight.
   - You will NOT be asked the encryption question: the project
     already declares `ITSAppUsesNonExemptEncryption: false`
     (standard HTTPS only).

## 5. TestFlight — internal testing (no review needed)

1. App Store Connect → TestFlight → **Internal Testing → "+"** →
   create a group ("Family"), add yourself (and Lusik) as testers —
   internal testers must be added as users under Users and Access
   first (role: App Manager or Developer is fine).
2. Install **TestFlight** from the App Store on your iPhone, accept
   the email invite, install Lusik & Sons.
3. Builds expire after 90 days; upload a new one anytime (repeat step 4).

**What to test on real hardware** (the things the simulator can't):
- Haptics: add to bag (medium), qty steps, tab taps, the order-success
  and waitlist-success buzzes
- The Liquid Glass island blur + lens glide over real content
- The immersive pill sheet: drag, flick, detent memory, breathe hint
  (first open only), photo tap → collapse → lightbox pinch-zoom
- Dark mode + a large Dynamic Type size (Settings → Accessibility)
- On an iPad (or when the Fold exists): the open-book spread — photos
  left page, buy column right page; paired-up journal/shop grids

> **⚠️ Testing checkout = a real charge.** The app uses the live
> backend and live Stripe. The honest way to test it end-to-end: place
> one real order to yourself (a $20 Hye Em Yes bib), watch the order
> land in `/admin` and both confirmation emails arrive, then refund it
> from the Stripe dashboard (Payments → ⋯ → Refund) — which also
> exercises the refund-notification email. Don't use Stripe test
> cards; they're declined on live keys.

**Optional — external TestFlight:** a public link for up to 10,000
testers. Needs Beta App Review (~1 day, much lighter than full
review). Skippable for a family shop; internal testing is enough.

## 6. Before the real submission — the privacy URL

The App Store listing requires a public privacy policy URL. The plan
of record: add a `/privacy` route to the website that renders the
existing Privacy Policy content (a small website PR on `main`, NOT
from this branch). Until that page exists, stop here — everything
above (TestFlight) works without it.

## 7. Fill in the listing

Open [`APP_STORE.md`](./APP_STORE.md) side-by-side and paste through
App Store Connect → App Information / Pricing / App Privacy / the
version page:

- Identity, category, age rating answers, description, keywords,
  support + marketing URLs → all in APP_STORE.md
- **App Privacy questionnaire**: answer EXACTLY per the table in
  APP_STORE.md (it mirrors `PrivacyInfo.xcprivacy`; mismatches are a
  rejection magnet)
- Screenshots: the 6.9" shot list in APP_STORE.md (capture with ⌘S in
  the iPhone 16 Pro Max simulator)
- Pricing: **Free**, all territories (or U.S.-only to match
  "U.S. shipping only" — your call; U.S.-only is the honest start)
- Select the processed build from step 4

## 9. What happens next (and the likely pushbacks)

Most reviews complete in 24–48 hours. If it's rejected, don't panic —
respond in Resolution Center; these are the three plausible ones:

- **"Guideline 3.1.1 — the app sells content without IAP."** The
  reviewer mistook the goods for digital. Reply: every product is a
  physical, hand-stitched textile shipped by mail; per 3.1.3(e),
  physical goods must NOT use IAP; checkout is Stripe's hosted page.
  (The review notes above usually pre-empt this.)
- **"Guideline 4.2 — minimum functionality / web wrapper."** Reply:
  the app is fully native SwiftUI — native navigation, gesture-driven
  product sheets, a zoomable photo viewer, persistent bag with haptic
  feedback, offline-tolerant journal — the ONLY web view is Stripe's
  payment page, which 3.1.3(e) expects.
- **"Guideline 5.1.1/5.1.2 — privacy."** Check the App Privacy answers
  match APP_STORE.md's table and that the `/privacy` URL loads
  publicly (no login, renders on a phone).

## 10. After approval

- **Release option:** choose "Manually release this version" when
  submitting — approval can land at 2 AM; you press Release when
  you're ready to share it.
- The app goes live on the store within a few hours of release.
- **Updates** repeat steps 4 → 7's build selection → 8: bump
  `MARKETING_VERSION` (1.0.1, 1.1.0…), bump `CURRENT_PROJECT_VERSION`,
  archive, upload, select build, submit. TestFlight keeps working
  between releases.
- Reply to App Store reviews from App Store Connect — answers show
  publicly under the review, very on-brand for a shop where Lusik
  answers herself.
